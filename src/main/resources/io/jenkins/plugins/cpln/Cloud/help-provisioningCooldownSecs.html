Specify the cooldown period in seconds between provisioning new agents when using unique agents.

Default: 60 seconds

Purpose: This setting prevents Jenkins from over-provisioning agents when using the "Use Unique Agents" option. Without this cooldown, Jenkins may provision multiple agents for a single job because the NodeProvisioner doesn't wait for agents to connect before requesting more.

How it works:

  • When a new agent is provisioned, a cooldown timer starts
  • Additional provision requests during the cooldown are skipped
  • The cooldown is cleared when an agent comes online
  • The plugin also checks for pending (not yet online) agents before provisioning

Recommendations:

  • Set this to roughly how long it takes for a CPLN workload to start and connect
  • 60 seconds is usually a good starting point
  • Increase if you see multiple agents being provisioned for single jobs
  • Set to 0 to disable cooldown (not recommended with unique agents)
