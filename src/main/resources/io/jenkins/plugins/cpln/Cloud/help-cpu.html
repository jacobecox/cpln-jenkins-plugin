Specify the CPU resources in millicores (m) for the Jenkins agent container.

Minimum: 50 millicores (required for Jenkins agent to start)

Recommended: 200 millicores or higher

Default: 200 millicores

Important: The Jenkins inbound-agent is a Java process that requires adequate CPU to start. Values below 50m will likely cause the agent to fail to start. Values below 200m may cause slow startup or poor performance.

Note: Do not exceed your CPLN organization's CPU quota.

Examples:

  • 50m = absolute minimum (not recommended)
  • 200m = recommended for light Jenkins agents
  • 500m = recommended for moderate build workloads
  • 1000m = 1 full CPU core for heavy builds
