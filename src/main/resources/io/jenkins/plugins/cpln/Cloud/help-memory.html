Specify the Memory resources in mebibytes (MiB) for the Jenkins agent container.

Minimum: 128 MiB (required for Jenkins agent to start)

Recommended: 256 MiB or higher

Default: 512 MiB

Important: The Jenkins inbound-agent is a Java process that requires adequate memory for the JVM heap. Values below 128Mi will likely cause the agent to fail to start. Values below 256Mi may cause OOM (Out of Memory) kills during builds.

Note: Do not exceed your CPLN organization's memory quota.

Examples:

  • 128 MiB = absolute minimum (not recommended)
  • 256 MiB = minimum recommended for Jenkins agents
  • 512 MiB = good for light to moderate builds
  • 1024 MiB = recommended for moderate to heavy builds
  • 2048 MiB = for memory-intensive builds (e.g., large Java projects)
